كورس ++C من الأساسيات إلى الكلاسات

شرح عربي/إنجليزي + أمثلة حقيقية + صفحة اختبار منفصلة

مقدمة بنية البرنامج المتغيرات والأنواع النطاق والعمر مؤشرات vs مراجع الذاكرة الديناميكية الكلاسات والتغليف الباني والهادم الوراثة Polymorphism ملفات الاختبار

1. مقدمة سريعة

الهدف من هالموقع إنك ما تضل على مستوى "cout و cin" فقط، بل تدخل على منطقة السيطرة الحقيقية على الذاكرة والكائنات، عشان لما تشوف كود دكتورك أو مشروع تخرجك ما تتفاجأ.

The goal now is to go beyond basic I/O and learn how C++ manages memory, objects, and abstraction. This is what makes C++ powerful (and a bit dangerous if you ignore memory).

2. بنية برنامج C++ ولماذا نكتب الأشياء بهذا الترتيب؟

أي برنامج C++ يمر بثلاث مراحل تقريبًا: المعالجة المسبقة (preprocessing) → الترجمة (compilation) → الربط (linking). لهذا السبب نكتب #include أولاً. 

#include <iostream>        // تجهيز مكتبة الإدخال والإخراج
using namespace std;      // اختصار std::

int main() {              // نقطة الدخول
    cout << "Hello";      // جسم البرنامج
    return 0;             // رجوع ناجح
}
The compiler needs to know the declarations before main(). That's why headers come first.

3. المتغيرات وأنماط التخزين

غير إن فيه أنواع (int, double, char) فيه كمان "طريقة التخزين": محلي (داخل دالة)، عام (global)، وثابت (const). 

int g = 10;          // متغير عام - متاح لكل الملف

int main() {
    const double PI = 3.14;   // ثابت لا يتغير
    int x = 5;                // محلي داخل main
}
Global variables live for the whole program. Local variables live only inside the block. const protects values from change.

4. نطاق المتغير (Scope) وعمره (Lifetime)

النطاق = فين تقدر تشوف المتغير. العمر = متى ينشأ ومتى يُدمّر. 

void test() {
    int a = 10;    // ينشأ هنا، ينتهي بنهاية الدالة
    {
        int b = 20;  // ينشأ هنا، يموت بنهاية القوس
    }
}
Block scope is important when you write large classes to avoid name conflicts.

5. المؤشرات مقابل المراجع (Pointers vs References)

هذه من أكثر الأشياء اللي تلخبط الطلاب. خلّينا نفصلها.

المؤشر متغير يخزن "عنوان" متغير آخر، ونقدر نغيره ليشير لشيء آخر.

int x = 10;
int *p = &x;   // p يشير إلى x
*p = 50;       // غيرنا x عن طريق المؤشر

المرجع (Reference) هو اسم ثاني لنفس المتغير، ولا يمكن إعادة ربطه.

int x = 10;
int &r = x;   // r هو x
r = 30;       // x صار 30
A pointer can be reassigned and can be null. A reference must be initialized and cannot be null. That's why references are safer for function parameters.

تمرير بالمرجع يوفر نسخ البيانات الكبيرة:

void swap(int &a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}

6. الذاكرة الديناميكية (Dynamic Memory) بـ new / delete

أحيانًا ما نعرف الحجم إلا وقت التشغيل (runtime) → نستخدم new لحجز ذاكرة و delete لتحريرها. 

int *p = new int;   // حجز int في الهيب
*p = 100;
cout << *p << endl;
delete p;           // لازم نمسح

لو بدنا مصفوفة حجمها يدخل من المستخدم: 

int n;
cin >> n;
int *arr = new int[n];

for (int i = 0; i < n; i++)
    cin >> arr[i];

// ... استعمال المصفوفة ...

delete[] arr;   // مهم جدا
If you allocate with new, you must deallocate with delete. If you allocate an array with new[], you must free it with delete[]. Otherwise you get a memory leak.

7. الكلاسات والتغليف (Encapsulation)

الفكرة: نخلي البيانات private، ونسمح بالوصول لها عن طريق دوال public. 

class BankAccount {
private:
    double balance;      // ما حدا يلمسه مباشرة
public:
    BankAccount() {      // باني افتراضي
        balance = 0;
    }
    void deposit(double amount) {
        if (amount > 0) balance += amount;
    }
    double getBalance() {
        return balance;
    }
};
This is encapsulation: protect the data, expose behavior.

8. الباني (Constructor) والهادم (Destructor)

الباني دالة تتنفّذ تلقائيًا عند إنشاء الكائن. الهادم يتنفّذ عند انتهاء عمره لتنظيف الموارد. 

#include <iostream>
using namespace std;

class FileHandler {
public:
    FileHandler() {
        cout << "Opening file..." << endl;
    }
    ~FileHandler() {
        cout << "Closing file..." << endl;
    }
};

int main() {
    FileHandler f;  // هنا يستدعي الباني
    // ... شغل ...
} // هنا يستدعي الهادم
Constructors initialize objects. Destructors clean up (especially important with dynamic memory or files).

9. الوراثة (Inheritance)

نستخدمها لما يكون عندنا كلاس عام (Person) ونبني عليه كلاس أكثر تحديدًا (Student). 

class Person {
public:
    string name;
    void sayHi() {
        cout << "Hi, I'm " << name << endl;
    }
};

class Student : public Person {
public:
    int id;
};
public inheritance means: "a Student is a Person". So it inherits public members.

10. تعدد الأشكال (Polymorphism) والـ virtual

هذا من أكثر الأشياء "شكلها غريب" لكن مفيدة. الفكرة: نتعامل مع المؤشر للـ base class، ونخلّي الدالة تتصرف حسب نوع الكائن الحقيقي. 

#include <iostream>
using namespace std;

class Shape {
public:
    virtual void draw() {   // virtual = نفّذ دالة الابن لو وُجدت
        cout << "Drawing shape..." << endl;
    }
};

class Circle : public Shape {
public:
    void draw() override {
        cout << "Drawing circle" << endl;
    }
};

int main() {
    Shape *s;        // مؤشر لكلاس عام
    Circle c;
    s = &c;
    s->draw();       // ينفّذ draw تبعة Circle مش Shape
    return 0;
}
This is runtime polymorphism. The call is resolved at runtime thanks to virtual.

11. التعامل مع الملفات (File I/O) – تمهيد

عشان تخزّن بياناتك خارج البرنامج، تستعمل ملفات. 

#include <fstream>
#include <iostream>
using namespace std;

int main() {
    ofstream out("data.txt");     // فتح ملف للكتابة
    out << "Hello from Qais" << endl;
    out.close();

    ifstream in("data.txt");      // فتح ملف للقراءة
    string line;
    while (getline(in, line)) {
        cout << line << endl;
    }
    in.close();
    return 0;
}
ofstream for writing, ifstream for reading, fstream for both.
